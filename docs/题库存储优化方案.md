# 心理测试网站 题库存储优化方案

## 1. 优化背景

### 1.1 当前问题
- 题库数据存储在后端数据库，同步机制复杂
- 前后台题目数据容易不一致
- 每次修改题目需要同步，流程繁琐
- 管理后台只控制上下线，但题目数据冗余

### 1.2 优化目标
- **题库全部存储在前端** - 所有题目数据保存在前端代码中
- **后台仅存储配置** - 后端只存储测试的基本信息（名称、图标、价格、上下线状态等）
- **简化管理后台** - 后台只控制测试的显示/隐藏，不修改题目内容

## 2. 优化方案

### 2.4 前端逻辑修改

#### 2.4.1 getQuizList() 简化
```javascript
// 从后端只获取配置信息
export async function getQuizList() {
  // 从API获取配置
  const configs = await fetchQuizConfigs()

  // 合并本地题库数据
  return quizzes.map(q => {
    const config = configs.find(c => c.code === q.code)
    return {
      ...q,  // 本地完整数据
      status: config?.status ?? 1,  // 从配置读取上下线
      price: config?.price ?? q.price,
      icon: config?.icon ?? q.icon,
      color: config?.color ?? q.color,
      paid: config?.paid ?? q.paid
    }
  }).filter(q => q.status === 1)  // 只返回上线的
}
```

#### 2.4.2 getQuizByCode() 简化
```javascript
export function getQuizByCode(code) {
  // 直接从本地获取，无需等待API
  const quiz = quizzes.find(q => q.code === code)
  if (!quiz) return null

  // 异步获取配置信息
  fetchQuizConfig(code).then(config => {
    if (config) {
      quiz.price = config.price
      quiz.status = config.status
      quiz.paid = config.paid
    }
  })

  return quiz
}
```

### 2.5 后端改造
