## 二、本次优化需求

### 2.1 热门推荐样式优化

**问题描述**：
- 首页热门推荐模块与顶部分类筛选样式重合
- 用户视觉体验不够清晰

**优化方案**：
- 重新设计热门推荐卡片样式
- 与分类标签区分开，突出热门标识
- 优化视觉层次感

---

### 2.2 管理平台安全优化

**问题描述**：
- 登录页面默认密码直接写在页面上
- 存在安全风险

**优化方案**：
- 移除页面硬编码的默认密码
- 后管首次登录强制要求修改密码
- 支持在后台设置页面修改管理密码

---

### 2.3 付费测试显示控制

**问题描述**：
- 测试结果显示"免费测"和"19.9解锁"重复
- 用户体验不够简洁

**优化方案**：
- 在后台管理添加"付费开关"
- 每个测试独立控制：免费显示/付费显示/全部显示
- 前端根据配置动态显示不同按钮
  - 免费模式：直接显示完整结果
  - 付费模式：显示解锁按钮引导付费
  - 全部模式：两种都显示

---

### 2.4 仪表盘数据真实性

**问题描述**：
- 后台仪表盘显示的是假数据
- 无法反映真实运营情况

**优化方案**：
- 真实数据存储到数据库
- 统计维度：
  - 总测试次数
  - 总用户数
  - 付费订单数
  - 收入金额
  - 每日/每周/每月趋势图
- 清除所有模拟数据

### 2.5 题库下线同步控制

**问题描述**：
- 后台下线测试后，前台立即生效，用户可能正在答题被中断
- 需要一个确认同步的流程

**优化方案**：
- 后台题库管理：下线的数据也正常显示（带下线标识）
- 后台操作流程修改：
  1. 后台点击"下线"按钮 → 测试状态变为"已下线"
  2. 新增"发布到前台"按钮 → 点击后同步到前台
  3. 前台才会显示/隐藏对应测试
- 前台数据加载优先级：
  1. 先检查是否有待发布的更新
  2. 如果有，使用本地缓存数据
  3. 如果没有本地数据，再从数据库获取
  4. 确保前台加载速度和数据一致性

### 2.6 报告免责声明配置

**问题描述**：
- 测试报告目前没有免责声明
- 法律风险和用户体验不完整

**优化方案**：
- 在后台管理系统添加"系统配置"功能
- 配置项：
  - 免责声明内容
  - 内容来源说明
  - 联系方式/商务合作
- 前端每个报告底部自动添加免责声明
- 后台修改后，前端实时同步更新

**技术实现**：
```javascript
// 系统配置模型
{
  _id: ObjectId,
  key: String,           // 'disclaimer' | 'content_source' | 'contact'
  value: String,         // 配置值
  updatedAt: Date
}
```
